S3. Model fitting workflow

This section documents the model-fitting workflow used in the study. It includes:

  • Location–scale models for bloom phenology (Gaussian outcomes)
  • Hierarchical Bayesian models for phenological mismatch (Gaussian) and reproductive performance (ordinal cumulative probit)

Most code chunks are provided as a reproducible workflow record (eval: false) rather than executed during rendering of the supplementary document.

Location–scale models of bloom phenology

These models tested whether both the mean timing and variability of bloom onset (days since the winter solstice) changed over time.

NoteVariable definitions

  • delta5: Bloom onset date (days since winter solstice) at the 5% threshold.

  • delta15: Bloom onset date (days since winter solstice) at the 15% threshold.

  • delta25: Bloom onset date (days since winter solstice) at the 25% threshold.

  • TIME: Calendar year, centered (not scaled).

Model function

A wrapper function was used to fit Gaussian location–scale models for each bloom threshold (5%, 15%, 25%), with TIME included in both the location (μ) and scale (σ) submodels.

run_location_scale_models <- function(response_vars, data) {
  if (!dir.exists(here("Rdata"))) dir.create(here("Rdata"))
  
  purrr::walk(response_vars, function(var) {
    model_formula <- bf(
      as.formula(paste(var, "~ 1 + TIME")),
      sigma ~ 1 + TIME
    )
    
    file_path <- here("Rdata", paste0("model_", var, ".rds"))
    model_family <- gaussian()
    model_priors <- get_prior(model_formula, data = data, family = model_family)
    
    fit <- tryCatch({
      brm(
        formula = model_formula,
        data = data,
        prior = model_priors,
        family = model_family,
        iter = 6000, warmup = 2000, chains = 4, seed = 123, init = 0,
        control = list(adapt_delta = 0.99, max_treedepth = 15),
        file = file_path,
        file_refit = "on_change"
      )
    }, error = function(e) {
      cat("Error fitting model for:", var, "\n")
      cat("Error message:", e$message, "\n")
      return(NULL)
    })
    
    if (!is.null(fit)) cat("Saved model for:", var, "to", file_path, "\n")
  })
}

Automated model fitting workflow

To systematically evaluate candidate models, we used a custom function (run_analysis_workflow()) that automates model fitting, storage, and model comparison.

The workflow:

  • iterates over candidate fixed- and random-effects structures,

  • supports both Gaussian and ordinal families,

  • optionally models auxiliary parameters (sigma for Gaussian models; disc for ordinal models),

  • skips models that were already fitted,

  • saves fitted models and leave-one-out cross-validation (LOO) objects, and

  • uses tryCatch() so individual failures do not halt the full workflow.

  fit_and_save_model <- function(name, mean_random, aux_formula, ...) {
    model_path <- file.path(output_dir, paste0("female_fit_", name, ".rds"))
    loo_path <- file.path(output_dir, paste0("female_loo_", name, ".rds"))
    
    if (file.exists(model_path)) {
      message(paste0("Skipping '", name, "': Model file already exists."))
      return(invisible(NULL))
    }
    
    message(paste0("\n--- Fitting model: ", name, " ---"))
    
    formula_str <- paste(response_var, "~", fixed_effects, "+", mean_random)
    
    # --- CORRECTED LOGIC ---
    # Use inherits() for a robust check if aux_formula is a real formula or NA.
    brm_formula <- if (!inherits(aux_formula, "formula")) {
      bf(formula_str, nl = FALSE)
    } else if (is_gaussian) {
      bf(formula_str, sigma = aux_formula, nl = FALSE)
    } else {
      bf(formula_str, disc = aux_formula, nl = FALSE)
    }
    
    model_priors <- get_prior(brm_formula, data = data, family = model_family)
    
    fit <- tryCatch({
      brm(
        formula = brm_formula, data = data, prior = model_priors, family = model_family,
        iter = 6000, warmup = 2000, chains = 4, seed = 123, init = 0,backend = "cmdstanr",threads = threading(12),
        control = list(adapt_delta = 0.99, max_treedepth = 15)
      )
    }, error = function(e) { message(paste0("ERROR fitting '", name, "': ", e$message)); NULL })
    
    if (!is.null(fit)) {
      saveRDS(fit, model_path)
      loo_result <- tryCatch(loo(fit), error = function(e) { message(paste0("ERROR with LOO: ", e$message)); NULL })
      if (!is.null(loo_result)) saveRDS(loo_result, loo_path)
    }
  }
  
  # --- Run all models and compare LOO results ---
  pwalk(model_specs, fit_and_save_model)

# Location–scale models of bloom phenology

These models tested whether both the mean timing and variability of bloom onset (days since the winter solstice) changed over time.

::: {.callout-note title="Variable definitions"}

- delta5: Bloom onset date (days since winter solstice) at the 5% threshold.

- delta15: Bloom onset date (days since winter solstice) at the 15% threshold.

- delta25: Bloom onset date (days since winter solstice) at the 25% threshold.

- TIME: Calendar year, centered (not scaled).
:::

## Model function

A wrapper function was used to fit Gaussian location–scale models for each bloom threshold (5%, 15%, 25%), with `TIME` included in both the location (μ) and scale (σ) submodels.

```{r}
#| label: bloom-function
#| eval: false
run_location_scale_models <- function(response_vars, data) {
  if (!dir.exists(here("Rdata"))) dir.create(here("Rdata"))
  
  purrr::walk(response_vars, function(var) {
    model_formula <- bf(
      as.formula(paste(var, "~ 1 + TIME")),
      sigma ~ 1 + TIME
    )
    
    file_path <- here("Rdata", paste0("model_", var, ".rds"))
    model_family <- gaussian()
    model_priors <- get_prior(model_formula, data = data, family = model_family)
    
    fit <- tryCatch({
      brm(
        formula = model_formula,
        data = data,
        prior = model_priors,
        family = model_family,
        iter = 6000, warmup = 2000, chains = 4, seed = 123, init = 0,
        control = list(adapt_delta = 0.99, max_treedepth = 15),
        file = file_path,
        file_refit = "on_change"
      )
    }, error = function(e) {
      cat("Error fitting model for:", var, "\n")
      cat("Error message:", e$message, "\n")
      return(NULL)
    })
    
    if (!is.null(fit)) cat("Saved model for:", var, "to", file_path, "\n")
  })
}
```

# Hierarchical Bayesian models of reproductive performance

## Data preparation

The primary breeding dataset was loaded and pre-processed separately for females and males.

Key preprocessing steps included:

Type conversion of predictors to numeric

Encoding reproductive outcome (`FLEDS`) as an ordered factor for cumulative ordinal models

Centering (not scaling) continuous predictors (`TIME`, `X5MISMATCH`, `AGE`, `AFR`, `LONGEVITY`)

Converting `YEAR` and `RING` to factors for use as grouping variables

::: {.callout-note title="Variable definitions"}

- `X5MISMATCH`: Phenological mismatch (days) using the 5% bloom threshold

- `FLEDS`: Number of fledged chicks (ordinal response: 0–3)

- `TIME`: Calendar year, centered (not scaled)

- `ZX5MISMATCH`: Centered phenological mismatch (5% threshold)

- `ABSMATCH`: Centered absolute mismatch (5% threshold)

- `AGE`: Age, centered

- `AFR`: Age at first reproduction, centered

- `LONGEVITY`: Age at last observation (proxy for lifespan), centered

- `YEAR`: Calendar year (factor)

- `RING`: Individual identifier (factor)
:::

# Automated model fitting workflow

To systematically evaluate candidate models, we used a custom function (`run_analysis_workflow()`) that automates model fitting, storage, and model comparison.

The workflow:

- iterates over candidate fixed- and random-effects structures,

- supports both Gaussian and ordinal families,

- optionally models auxiliary parameters (sigma for Gaussian models; disc for ordinal models),

- skips models that were already fitted,

- saves fitted models and leave-one-out cross-validation (`LOO`) objects, and

- uses `tryCatch()` so individual failures do not halt the full workflow.

```{r}
      brm(
        formula = brm_formula, data = data, prior = model_priors, family = model_family,
        iter = 6000, warmup = 2000, chains = 4, seed = 123, init = 0,backend = "cmdstanr",threads = threading(12),
        control = list(adapt_delta = 0.99, max_treedepth = 15)
      )
    }, error = function(e) { message(paste0("ERROR fitting '", name, "': ", e$message)); NULL })
    
    if (!is.null(fit)) {
      saveRDS(fit, model_path)
      loo_result <- tryCatch(loo(fit), error = function(e) { message(paste0("ERROR with LOO: ", e$message)); NULL })
      if (!is.null(loo_result)) saveRDS(loo_result, loo_path)
    }
  }
  
  # --- Run all models and compare LOO results ---
  pwalk(model_specs, fit_and_save_model)
  

}
```
